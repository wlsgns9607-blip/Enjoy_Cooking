# 요리가 즐거워! — React 프로토타입

제철 식재료 지도 탐색, 대화형 AI 레시피 코칭, 그리고 요리일지 중심의 홈쿡 기록/성장형 미식 플랫폼의 화면 프로토타입입니다. Stitch로 추출한 와이어프레임을 기반으로, 실제 작동하는 React 코드로 완성도 있게 구현했습니다.

## 💡 기존 요리/레시피 앱과의 차별점

본 프로젝트는 단순 레시피 리스트 검색 기능을 넘어, 사용자의 능동적인 홈쿡 여정을 돕기 위해 다음과 같은 핵심 차별성을 제공합니다.

1. **시각적인 지역·시즌 제철 식재료 탐색 (Map)**
   - 기존의 일반적인 텍스트 카테고리 검색 대신, 전국 지도를 활용해 현재 시즌의 지역별 제철 특산품 및 식재료를 직관적으로 탐색하고 즉시 AI와 연동하여 요리 영감을 얻도록 유도합니다.

2. **실시간 맞춤 대화형 레시피 코칭 (AI Chef)**
   - 박제된 텍스트 레시피가 아닌, 실시간 AI 셰프와 대화하며 식재료 보관법, 대체 재료, 양념장 팁 등을 묻고 구체적인 맞춤형 피드백을 받아 조리할 수 있습니다.

3. **성취감 중심의 홈쿡 아카이빙 (Diary & Badge)**
   - 직접 한 요리를 사진 일지 형태로 감성적으로 기록하고, 활동 성취에 따른 칭호와 뱃지 획득을 통해 건강한 요리 습관을 형성하도록 리워드를 제공합니다.

4. **단절 없는 유기적 서비스 여정 (User Journey Loop)**
   - `지도(재료 발견) ➔ AI 코칭(레시피 탐색 및 문답) ➔ 요리일지(성취 아카이브) ➔ 커뮤니티(소통 및 공유)`로 이어지는 순환 고리를 형성하여 사용자의 높은 서비스 인게이지먼트를 확보합니다.

## 👥 타겟 페르소나 (Target Persona)

사용자 중심의 AX(AI Experience) 설계 및 핵심 기능(지도, AI 코칭, 일지) 고도화를 위해 정의한 3가지 핵심 페르소나입니다.

### 1. 최지현 (46세, 피부과 원장) - "건강하고 기분 좋은 미식을 즐기는 전문직 미식가"
* **환경/배경**: 서울 서대문구 거주. 바쁜 일상 속에서 제철 음식을 챙겨 먹고 싶지만, 시장 갈 시간이 없고 어떤 식재료가 진짜 제철인지 일일이 확인하기 번거로워함.
* **목표 (Goal)**: 건강하고 품격 있는 '제철 미식'을 간편하게 집에서 즐기는 것.
* **현재 해결 방식**: 백화점 식품관 이용, 맛집 검색.
* **사용 시나리오**: 퇴근길 혹은 주말 저녁 여유가 있을 때 앱 실행 ➔ **[대한민국 제철 지도]**에서 지금 가장 맛이 오른 지역별 식재료 정보를 직관적으로 탐색 ➔ 나만의 레시피로 요리한 뒤 그 노하우를 **[요리 일지]**에 기록.
* **기대 효과**: 제철 식재료를 깊이 있게 즐기는 미식가로서의 정체성을 강화하고, 커뮤니티를 통해 초보자들에게 노하우를 나누는 멘토 역할 수행.

---

### 2. 박영민 (32세, 수입차 딜러) - "배달보다 건강하고 뒷정리가 깔끔한 가성비 집밥을 원하는 직장인"
* **환경/배경**: 대전 서구 거주. 잦은 야근과 출장으로 배달 음식을 자주 먹어 몸이 무거워짐을 느낌. 요리를 해보고 싶으나 퇴근 후에는 체력이 방전되어 '재료 손질'과 '뒷정리'에 부담을 느낌.
* **목표 (Goal)**: 손질 과정이 최소화되고 설거지 등 뒤처리가 간편하면서도 건강을 챙길 수 있는 가성비 한 끼.
* **현재 해결 방식**: 배달 앱, 편의점 도시락.
* **사용 시나리오**: 퇴근 직전 차 안이나 귀가 30분 전 앱 실행 ➔ **[AI 요리박사]**에게 *"오늘 너무 피곤한데, 손질할 게 거의 없는 제철 배추 요리 추천해줘"*라고 문답 ➔ AI가 제안한 전자레인지를 활용한 간편 배추찜 등 '설거지 최소화 레시피'로 조리.
* **기대 효과**: 요리가 무거운 가사 노동이 아닌, 배달 음식보다 저렴하고 건강하게 즐길 수 있는 30분 내외의 가벼운 라이프스타일 취미로 전환.

---

### 3. 박서영 (25세, 대학생) - "실패를 두려워하지 않고, 요리 일지로 성장을 기록하는 새내기 자취러"
* **환경/배경**: 인천 계양구 거주. 이제 막 요리에 흥미를 붙이기 시작한 자취생. 재료 손질부터 보관 요령까지 모르는 것이 많아 실패에 대한 두려움이 큼.
* **목표 (Goal)**: 요리하는 즐거움과 성장하는 전 과정을 기록하고 타인과 활발히 공유하기.
* **현재 해결 방식**: 유튜브 요리 채널 시청, 요리 블로그 레시피 참고.
* **사용 시나리오**: 자취방에서 친구들을 초대하거나 요리를 연습할 때 앱 실행 ➔ 요리 과정 중 막히는 부분이 있을 때 **[AI 요리박사]**에게 질문하여 실시간 코칭을 받음 ➔ 완성 후 사진을 촬영하여 **[요리 일지]**를 남기고 뱃지를 획득.
* **기대 효과**: 일지를 채우는 아카이빙 재미와 등급/뱃지 시스템을 통해 자연스럽게 요리 실력을 레벨업하고, 커뮤니티 내 '방구석 셰프' 고수들과 활발히 소통.

## 실행 방법

```bash
npm install
npm run dev
```

터미널에 나오는 주소(보통 `http://localhost:5173`)를 브라우저로 열면 됩니다. 빌드가 필요하면 `npm run build`를 사용하세요.

## 기술 스택

- React 18 + Vite
- 아이콘: [lucide-react](https://lucide.dev/)
- 폰트: Pretendard (CDN)
- 별도 UI 라이브러리 없이 순수 CSS(`src/App.css`)로 디자인 토큰을 관리합니다.

## 폴더 구조

```
src/
  App.jsx              화면 전환·모달 상태를 들고 있는 루트 컴포넌트
  App.css              디자인 토큰(색상/타이포/라운드) 및 전체 스타일
  components/
    BottomNav.jsx       하단 탭 4개 (지도 / AI 요리박사 / 요리일지 / 마이페이지)
    MapScreen.jsx        제철 지도 화면
    AiScreen.jsx         AI 코칭 + 커뮤니티 (세그먼트 탭으로 전환)
    DiaryScreen.jsx      요리일지 (데이터가 비면 자동으로 빈 상태 화면 표시)
    ProfileScreen.jsx    마이페이지 (등급, 진행률, 뱃지)
    ConnectModal.jsx     계정 연동 팝업 모달
```

## 핵심 설계 결정 (디자이너·개발자 인계 시 같이 전달할 것)

1. **로그인 화면이 없습니다.** 앱 진입 장벽을 낮춰 유입을 늘리는 게 목적입니다. 실제 서비스에서는 첫 실행 시 익명 식별자를 자동 발급해 요리일지·뱃지·커뮤니티 글을 거기에 귀속시키는 구조를 권장합니다. 이 프로토타입에서는 `ProfileScreen.jsx`에 닉네임이 하드코딩되어 있는데, 실제 구현 시 그 자리를 익명 식별자 발급 로직으로 교체하면 됩니다.
2. **계정 연동은 마이페이지에서만, 팝업으로.** 화면 전환 없이 `ConnectModal`이 현재 화면 위에 오버레이로 뜹니다. 강제 가입이 아니라 "기기를 바꿔도 기록을 지키고 싶을 때"의 선택 사항입니다.
3. **AI 코칭은 텍스트 입력만 사용합니다.** 카메라/이미지 인식 기능은 의도적으로 제외했습니다. 구현 난이도(인식 정확도, 응답 속도) 때문에 MVP 범위에서 제외한 것이며, 추후 고도화 버전 후보입니다.
4. **"재료 구매하기"는 다음 버전 기능입니다.** 산지 직송 커머스와 연결되는 기능으로, `AiScreen.jsx`에 주석으로 표시해두고 이번 MVP 코드에는 포함하지 않았습니다.
5. **포인트/재화 시스템(예: 화면 상단 고정 포인트 표시)은 포함하지 않았습니다.** 기획서에 없던 요소라 제외했습니다. 단, 마이페이지의 "다음 레벨까지 150P" 같은 레벨업 진행률은 원래 뱃지·등급 시스템의 일부이므로 유지했습니다.

## 📊 데이터 트래킹 및 서비스 고도화 계획 (PM 관점)

추후 실제 서비스 백엔드가 구축될 시, 다음과 같은 핵심 데이터 트래킹을 통해 추천 알고리즘을 고도화하고 사용자 경험을 최적화할 계획입니다.

* **핵심 질문**: *"추후 백엔드가 구현된다면 어떤 데이터를 트래킹할 것인가요?"*
* **답변 및 데이터 활용 방안**:
  - **식재료 관심도 측정**: 지도 화면에서 유저들이 가장 많이 클릭하고 탐색한 지역별/시즌별 식재료 데이터 수집.
  - **요리 의도 분석**: AI 요리박사 화면에서 사용자가 직접 검색하거나 질문한 요리 관련 주요 키워드 및 대화 텍스트 분석.
  - **추천 모델 고도화**: 수집된 식재료 관심도와 요리 키워드 데이터를 매칭 및 분석하여, 다음 주차 또는 다음 시즌 메인 화면에 우선 노출할 **'인기 제철 레시피' 추천 모델**을 고도화하고 사용자 인게이지먼트를 극대화할 예정입니다.
